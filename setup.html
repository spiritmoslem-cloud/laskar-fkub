1 Struktur file proyek

Aplikasi ini terdiri dari beberapa file yang harus berada dalam satu folder:

kardiodoc/
├── index.html # Aplikasi utama (publik untuk user)
├── admin.html # Editor admin (SEMBUNYIKAN — lihat §3)
├── data.js # Data 145 obat — di-update oleh admin
├── logo.js # Logo LASKAR (embedded base64)
├── styles.css # Styling utama
├── admin-styles.css # Styling admin
├── shared.jsx # Komponen bersama (modals, tags)
├── variant-a.jsx # Tampilan Clinical Refined
├── admin.jsx # Logic admin editor
├── tweaks-panel.jsx # (development tool, bisa dihapus untuk prod)
└── setup.html # Panduan ini

2 Deploy ke GitHub Pages

  1. Buat repo baru di GitHub (mis. kardiodoc). Bisa public atau private.
  2. Upload semua file ke repo (drag-drop di GitHub web, atau pakai git push).
  3. Settings → Pages → Source: main branch / root → Save.
  4. Tunggu 1–2 menit, app Anda akan online di https://username.github.io/kardiodoc/
Penting: GitHub Pages selalu publik. Jika repo private, kontennya tetap bisa diakses orang yang tahu URL — file source code akan bisa di-view via DevTools.

3 Proteksi data admin

Karena ini static HTML, ada 3 tingkat proteksi tergantung tingkat kerahasiaan:

🔒 Paling aman

Edit lokal, jangan deploy admin

Simpan admin.html hanya di laptop Anda. Edit lokal (buka file di browser), export data.js, commit ke GitHub. User tidak pernah lihat admin.

🛡️ Semi-privat

Sembunyikan URL admin

Letakkan admin di folder rahasia: /admin-kdc-2025-xyz/admin.html. Tidak terindex Google, hanya yang tahu URL bisa akses.

✓ Cara cepat

Netlify password protection

Deploy ke Netlify (gratis), aktifkan Site Settings → Visitor Access → Password protect. Tidak perlu kode.

Cara A: Edit lokal saja (Rekomendasi)

  1. Clone repo ke laptop Anda
  2. Buka admin.html langsung dengan double-click (atau drag ke browser)
  3. Edit obat → klik Export → Download data.js
  4. Replace data.js di folder repo dengan file baru
  5. Commit & push: 
    git add data.js
git commit -m "Update dosis Bisoprolol"
git push
  6. GitHub Pages otomatis rebuild — user dapat update dalam 1–2 menit

Cara B: Hapus admin.html sebelum deploy

Pada saat upload ke GitHub, jangan sertakan admin.html dan admin.jsx, admin-styles.css. Edit dilakukan di clone lokal.

Buat file .gitignore:

admin.html
admin.jsx
admin-styles.css

Cara C: Netlify password protection

  1. Buat akun di netlify.com (gratis)
  2. Drag folder proyek ke Netlify dashboard, atau connect ke repo GitHub
  3. Site settings → Visitor Access → Set password
  4. Akses your-site.netlify.app/admin.html akan minta password sebelum bisa lihat
Yang tidak melindungi data: password gate di admin.html hanyalah UX guard — orang yang paham DevTools bisa bypass dengan hapus localStorage atau jalankan JS langsung. Untuk proteksi nyata, kontrol siapa yang bisa akses URL admin (cara A, B, atau C di atas).

4 Cara update data obat

Action Cara
Tambah obat baru Admin → pilih kelompok → tombol "Tambah obat" di bawah daftar
Edit dosis Admin → klik obat → ubah field → otomatis tersimpan di browser
Reorder obat dalam kelompok Drag-and-drop di sidebar admin
Hapus obat / kelompok Tombol 🗑️ di sidebar (hover untuk muncul)
Bulk update banyak obat Admin → Import → drop file JSON atau CSV
Backup sebelum edit besar Admin → Export → Download JSON
Deploy update ke user Export data.js → ganti di repo → commit & push

Format CSV untuk bulk import

Header kolom (baris pertama CSV):

group_id,group_label,group_tone,n,b,s,t,rn,pd,pregCat,pregCls,pregTxt,c,se,tg,fn
  • Wajib: group_id, group_label, n (nama), s (dosis awal)
  • tg: tag indikasi dipisah dengan |, mis. HF|HT|CD
  • fn: 1 jika masuk Fornas, 0 jika tidak
  • tone: pilih dari blue, green, amber, rose, teal, slate, violet, orange, red

5 Update otomatis di HP user (PWA)

Aplikasi KardioDoc dirancang sebagai Progressive Web App (PWA) — bisa di-install di HP tanpa Play Store/App Store.

Bagaimana user install:

  • Android (Chrome): menu titik tiga → "Add to Home Screen" → app muncul di home seperti app native
  • iOS (Safari): tombol Share → "Add to Home Screen"
  • Setelah ter-install, app bisa berjalan offline (cache HTML/JS/data)

Bagaimana update menyebar:

  1. Anda push data.js baru ke GitHub
  2. User buka app di HP — browser cek server di background
  3. Jika ada perubahan, file baru di-fetch otomatis
  4. User akan lihat versi baru saat refresh berikutnya (biasanya dalam jam yang sama)
Tips: Jika ingin user langsung tahu ada update, tambah notifikasi visual seperti "Versi baru tersedia, refresh untuk update". Saya bisa bantu buat fitur ini jika diperlukan.

Apakah user bisa edit aplikasi mereka?

Tidak. User hanya membaca data. Walaupun mereka:

  • Save HTML ke file dan edit dengan text editor — hanya copy lokal mereka yang berubah, tidak affect orang lain
  • Akses admin.html (jika tidak Anda sembunyikan) — perubahan hanya tersimpan di localStorage browser mereka sendiri
  • Bypass password gate — sama, hanya local data yang berubah, master data Anda di GitHub aman

Data master Anda di GitHub tidak bisa diubah kecuali punya write access ke repo Anda.

6 FAQ

Saya lupa password admin?

Di password gate, klik link "Reset password". Atau buka DevTools (F12) → Application → Local Storage → hapus key kd_admin_pw dan kd_admin_pw_set.

Bookmark user akan hilang saat update?

Tidak. Bookmark tersimpan di localStorage di HP user, terpisah dari data obat. Tetap aman walaupun data.js di-update.

Beda admin.html dan index.html?

  • index.html = aplikasi untuk user (baca dosis, search, bookmark, dark mode)
  • admin.html = editor data untuk Anda (tambah/edit/hapus obat, export data.js)

Bisa multi-admin?

Karena ini client-side, setiap admin punya copy localStorage masing-masing. Mereka harus koordinasi siapa yang export data.js ke GitHub. Untuk multi-admin real-time, perlu backend (di luar scope project ini).

Versi modul lain di roadmap?

Roadmap aplikasi (lihat tombol Changelog) menyertakan kalkulator klinis, protokol titrasi HF, interaksi obat lengkap. Semua bisa ditambah sebagai modul terpisah saat dibutuhkan.
